Overview

Our goal in this project is to simulate the formation and trajectory of Saturn's rings using unity. Specifically, we model the gravitational acceleration and tangent velocity applied to the dust particles that result in their circuler trajectory. The collisions and gravitational attractions between dust particles are simulated with N-Body algorithm.

Progress

Since the data and the units of the formulas in astrophysics are very large, it is very difficult to use the real data completely in unity. In addition, there are some data that we have no way to know, such as the mass of each particle in the planetary ring. So we applied the formulas, idealized and scaled down the data and units, and finally made the initial process in unity that the particles can orbit Saturn and form a single planetary ring Our code is still in a relatively naive simulation stage. Our current code right now is basically based around how to apply astrophysical formulas. Then a certain number of blobs are generated to simulate the particles on Saturn's rings. This is of course an extremely inefficient method, and we will use N-body to improve the efficiency.

N-Body Modeling

Currently we use a double loop to check collisions between every pair of dust particles. This naive collision detection method is a very inefficient and costly, with which our computer could only support around 2000 particles to form a single layer of the planetory ring. In order to generate a more realistic multi-layer ring, our next step is to implement N-body collision simulation. We make reference to “REBOUND: an open-source multi-purpose N-body code for collisional dynamics”, which proposes a structure Octree that devides particles hiearchically. Since distant particles contribute less gravitational force than those nearby, we could separate those far away particles into groups that could be approximated with the total masses and center of masses. Only the gravitational forces of those particles nearby will be calculated individually. After applying the new collision detection algorithm, we will be able to decrease the runtime from O(N^2) to O(NlogN).

Future Plan

In addition to implementing Octree, we plan to expand the particle types to take account of the differences in mass and size. Shading and texture would also be added to simulate the color and texture of the Saturn.
